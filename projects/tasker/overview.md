# tasker

个人任务管理桌面 app：管任务 + 记时间 + 按日期/优先级组织每日工作节奏。

## 目标

- 单人本地使用，不做多端同步
- 任务的生命周期由**时间记录**驱动（不是靠手动改状态）
- 每天开工前能一眼看到"今天该做什么"，收工时能看到"今天做了什么"
- 磁盘存储对人友好：可用文本编辑器直接看/改

## 领域模型

### 任务 Task
- 标题
- 描述：markdown/富文本（大字段，单独文件）
- 分类：日常跟进 / 会议 / 我的任务 / Platform·Chaos test / 其他
- 状态：从时间记录汇总计算，不独立存储
  - 无时间记录 → 未开始
  - 有开始未结束 → 进行中
  - 最后一条时间记录带 `完成` 标志 → 完成
  - 最后一条带 `再开始` 标志 → 回到进行中

### 任务集合 TaskCollection + Membership
- **当前集合**：不带日期的活跃工作面
- **某天集合**：带日期
- **Backlog（虚拟）**：`membership == nil && status != .done`

任务通过 **Membership**（关联对象）归属集合。同一时刻只能属于一个集合。
- `kind`：`.current` 或 `.day(Day)`
- `priority`：今天必达 / 重要 / 正常 —— **属于关联，不属于任务本身**

任务归属和优先级都通过列表**右键菜单**操作，不作为详情里的常规属性编辑（除详情顶部的优先级快捷编辑外）。

### 时间记录 TimeEntry
- 标题（这段活是干什么的）
- 类型（工作类型）
- 开始时间 / 结束时间（都可编辑）
- 可选阶段性标志：`完成` / `再开始`

## 技术选型

- **平台**：macOS Native（默认 Swift + SwiftUI；未来跨平台再议）
- **存储**：本地磁盘
  - jsonl 存结构化数据（任务元信息、时间记录）
  - markdown 存描述等大字段（每个任务一个 `.md` 文件）
  - 默认根目录 `~/Documents/tasker/`；用户可在设置里改，路径存 `UserDefaults`（key `tasker.dataRoot`）
- **代码位置**：`app/tasker/`（SwiftPM 工程，与本项目文档目录 `projects/tasker/` 平级）
- **内存模型**：
  - 启动时全量加载
  - 内存改动 → 同步写盘（写盘前先校验磁盘版本，避免覆盖手改）
  - 定时扫盘（~10s）把外部手改同步回内存
- **代码风格**：DDD（聚合根 = Task；TimeEntry 作为 Task 内实体；TaskCollection 是查询/视图）

## UI 布局

- 左侧栏：任务列表，过滤器 = 某天/当天 · 当前 · Backlog
- 右侧：任务详情（markdown 编辑）+ 时间记录列表

## 明确不做

- 云同步 / 多端
- 多人协作 / 共享
- 通知 / 提醒（先不做，之后可加）
- 手动改任务状态（状态永远是时间记录算出来的）
